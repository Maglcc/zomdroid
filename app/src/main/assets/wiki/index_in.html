Bagaimana cara mendapatkan file game?

1. Unduh file game dari sumber resmi

Panduan untuk DepotDownloader

Zomdroid memerlukan file Project Zomboid versi Linux untuk berjalan. Cara termudah untuk mendapatkannya di Windows adalah melalui DepotDownloader — alat yang memungkinkan Anda mengunduh build tertentu langsung dari Steam.

Anda harus memiliki Project Zomboid di Steam sebelum mengunduh.

  1. Unduh DepotDownloader. Untuk Windows, ambil DepotDownloader-windows-x64.zip.
  2. Ekstrak ke mana saja, misalnya: C:\DepotDownloader
  3. Buka folder, klik kanan → Buka di Terminal / CMD
  4. Jalankan perintah berikut sebagai dasar, ganti NAMAPENGGUNA dan KATASANDI dengan kredensial Steam Anda:

    ./DepotDownloader -app 108600 -depot 108603 -manifest NOMERMANIFEST -username NAMAPENGGUNA -password KATASANDI

    Perintah lengkap termasuk nomor manifest dan flag -beta opsional (misalnya -beta unstable atau -beta legacy_42_12) dapat disalin langsung dari SteamDB — pilih DepotDownloader dari menu "Copy format" dan klik ikon salin di samping build yang diinginkan. Autentikasi Steam Mobile mungkin diperlukan.

2. Tambahkan file ke arsip .zip

Zomdroid memerlukan arsip .zip yang berisi file game di level teratas — bukan di dalam folder.

Panduan untuk Steam

Navigasi ke folder yang berisi file game (folder dengan executable ProjectZomboid64). Di Linux kemungkinan besar berada di ~/.local/share/Steam/steamapps/common/ProjectZomboid/projectzomboid. Di Windows atau macOS, perhatikan direktori unduhan depot dari langkah sebelumnya (akan muncul di konsol setelah unduhan selesai). Misalnya di Windows biasanya C:\Program Files (x86)\Steam\steamapps\content\app_108600\depot_108603\projectzomboid. Pilih semua file (Ctrl + A atau Command + A di MacOS) dan tambahkan ke arsip .zip

3. Salin arsip .zip ke perangkat mobile Anda

Salin arsip .zip dari langkah sebelumnya ke folder mana saja di perangkat mobile Anda. Anda dapat menggunakan file tersebut nanti untuk membuat instans game baru. Setelah instans game dibuat, Anda dapat menghapus arsip .zip untuk mengosongkan ruang.

Library native (multiplayer Build 41)

Untuk menggunakan multiplayer di Build 41, diperlukan library native tambahan. Library ini didistribusikan secara resmi oleh pengembang Project Zomboid sebagai bagian dari Build 42.12 (versi Linux).

⚠️Penting: file yang diperlukan hanya berada di dalam folder android. Hanya file dari folder ini yang boleh digunakan — file lain bukan library native.

Hanya dua library native yang diperlukan:

  • libRakNet64.so
  • libZNetNoSteam64.so

Unduh file-file ini dari distribusi Linux Build 42.12, tempatkan kedua file ke dalam satu arsip .zip, dan unggah arsip ini saat membuat instans game baru.

Arsip .zip harus berisi dua file .so di level teratas (bukan di dalam folder).

Anda dapat mengunggah arsip library native bersama dengan file game utama di kolom yang sesuai saat membuat instans game baru.

Renderer dan driver

Zomdroid menggunakan renderer untuk menerjemahkan grafis desktop Project Zomboid ke grafis yang kompatibel dengan perangkat Android.

Launcher menyediakan dua solusi rendering:

  • GL4ES
  • ZINK (ZINK_ZFA / ZINK_OSMESA)

GL4ES

GL4ES adalah renderer paling stabil untuk Build 41. Namun, di Build 42 sering menghasilkan artefak grafis, karena tidak dapat sepenuhnya menerjemahkan pipeline rendering game yang diperbarui.

ZINK

ZINK memberikan hasil terbaik dengan Build 42 dan biasanya menghilangkan artefak grafis yang terlihat dengan GL4ES.

ZINK menawarkan beberapa driver Vulkan:

  • Driver Vulkan Sistem
  • Freedreno
  • Turnip

Semua driver Freedreno dan Turnip yang tercantum di atas adalah build dari driver Vulkan Mesa open-source yang sama untuk GPU Adreno. Turnip adalah nama resmi komponen Vulkan dalam stack driver Mesa Freedreno — nama-nama tersebut kadang digunakan secara bergantian. Build yang berbeda dioptimalkan untuk generasi Snapdragon yang berbeda, itulah mengapa launcher menawarkan beberapa opsi driver.

ZINK terutama bekerja pada perangkat Snapdragon (GPU Adreno). Pada MediaTek Dimensity, ZINK mungkin bekerja melalui driver grafis ANGLE, namun mengaktifkan ANGLE pada perangkat Dimensity sering kali tidak mudah — jarang berhasil hanya melalui pengaturan pengembang dan mungkin memerlukan alat pihak ketiga. Pada MediaTek Helio, ZINK tidak didukung, tetapi beberapa perangkat Helio mungkin dapat menjalankan versi Build 42 yang lebih awal dan tidak terlalu berat (seperti 42.6) melalui GL4ES — belum ada solusi yang andal, tetapi ini mungkin berubah di masa depan. Untuk prosesor lain (Exynos, Kirin, dll.), informasi yang tersedia terbatas — ZINK sering tidak bekerja atau mungkin crash.

Konfigurasi yang direkomendasikan untuk Build 41 adalah GL4ES. Untuk Build 42, konfigurasi yang direkomendasikan adalah ZINK + driver Freedreno. Driver Freedreno hanya didukung pada perangkat Snapdragon dengan GPU Adreno. Versi driver yang berbeda menargetkan generasi hardware yang berbeda:

  • Freedreno — stabil, direkomendasikan untuk Adreno 6xx dan 7xx (Snapdragon hingga 7 Gen 1)
  • Freedreno 8xx — eksperimental, untuk Adreno 8xx (Snapdragon 8 Gen 3 / Gen 4 / Gen 5 / Elite)
  • Freedreno 830/840 — eksperimental, khusus untuk Adreno 830/840 (Snapdragon 8 Elite / Elite 2, misalnya seri Galaxy S25/S26)

Masalah yang diketahui

Build 42.12+ — crash saat pembuatan dunia

Mulai sekitar Build 42.12, game mungkin crash saat pembuatan dunia — khususnya saat memuat ke dalam game setelah pembuatan karakter. Ini tampaknya merupakan masalah di sisi game yang tidak terkait dengan perangkat atau driver, karena telah diamati pada berbagai perangkat dan konfigurasi.

Ini bukan kegagalan permanen. Jika game crash pada tahap ini, cukup restart Zomdroid dan coba lagi. Sebagian besar pengguna membutuhkan 2–4 percobaan sebelum dunia berhasil dimuat untuk pertama kalinya. Setelah dunia dibuat, pemuatan selanjutnya pada simpanan yang sama biasanya stabil.

Kompatibilitas mod grafis

Beberapa mod grafis — seperti retekstur pakaian, rambut, atau kendaraan — mungkin tidak ditampilkan dengan benar atau mungkin tidak berfungsi sama sekali. Ini adalah keterbatasan yang diketahui dari lapisan terjemahan grafis yang digunakan untuk menjalankan game di Android, dan tidak spesifik pada perangkat atau driver tertentu.

GUI menghilang di dekat kendaraan yang dibongkar

UI game mungkin menghilang jika ada kendaraan di dekatnya dengan 3 atau lebih roda yang dilepas. Penyebabnya adalah masalah rendering yang dipicu oleh kondisi kendaraan. Solusi: hindari melepas lebih dari 2 roda dari satu kendaraan. Jika UI sudah menghilang, menjauh dari kendaraan atau memuat ulang simpanan seharusnya memulihkannya.
